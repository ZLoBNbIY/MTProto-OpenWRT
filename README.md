Скрипт автоустановки MTProto прокси-сервера (mtg) на OpenWrt
Этот скрипт предназначен для быстрого развертывания легковесного прокси-сервера MTProto на базе mtg (v2) с поддержкой FakeTLS на роутерах под управлением OpenWrt.

Скрипт полностью автоматизирует процесс установки бинарных файлов, генерации секретных ключей, настройки брандмауэра (файрвола) и добавляет удобную панель управления в стандартный веб-интерфейс роутера (LuCI).

🚀 Основные возможности
Автоматическое определение архитектуры: Скрипт считывает архитектуру процессора роутера (aarch64, armv7, mipsel, mips, amd64) и скачивает под неё подходящий бинарный файл mtg.
FakeTLS-маскировка: Генерирует криптографический секретный ключ (начинающийся на ee), который маскирует Telegram-трафик под обычные HTTPS-запросы к доверенному домену (например, vk.com, google.com), защищая прокси от обнаружения провайдером (систем DPI).
Динамический файрвол: Автоматически создает правила в брандмауэре OpenWrt (через UCI):
Если выбран порт 443, настраивается перенаправление портов (Redirect WAN 443 -> Local 8443), чтобы избежать конфликта с веб-интерфейсом роутера LuCI (uhttpd).
Для других портов открывается прямой доступ.
При остановке или удалении службы все правила файрвола автоматически вычищаются.
Интеграция с LuCI: Устанавливает полноценное веб-приложение для управления в панель роутера (Службы ➔ MTProto Proxy), написанное по стандартам последних версий LuCI (на чистом JS):
Включение / выключение службы в один клик.
Быстрая смена порта и домена FakeTLS.
Генерация нового секрета одной кнопкой.
Отображение готовых ссылок для Telegram с учетом внешнего IP роутера.
📥 Инструкция по установке
Зайдите на свой роутер по SSH и выполните следующую команду:

bash
wget -qO- https://raw.githubusercontent.com/ZLoBNbIY/MTProto-OpenWRT/main/setup_mtproto.sh | sh
NOTE

Скрипт спросит входящий порт (по умолчанию 443) и домен маскировки (по умолчанию google.com).
По завершении установки вы получите готовую ссылку для подключения прямо в терминале, а в веб-интерфейсе LuCI появится вкладка управления.
⚙️ Управление через CLI (консоль SSH)
Служба прокси-сервера зарегистрирована в системе инициализации procd. Вы можете управлять ей стандартными командами:

Запустить прокси:
bash
/etc/init.d/mtg start
Остановить прокси:
bash
/etc/init.d/mtg stop
Перезапустить прокси:
bash
/etc/init.d/mtg restart
Включить автозапуск:
bash
/etc/init.d/mtg enable
Отключить автозапуск:
bash
/etc/init.d/mtg disable
Просмотр логов работы:
bash
logread | grep mtg
